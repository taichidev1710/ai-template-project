# 18 · Internationalization (đa ngôn ngữ)

i18next + react-i18next. Thiết lập trong `src/shared/config/i18n.ts`; chuỗi nằm
trong `src/locales/<lng>/common.json` (`vi`, `en`).

## Quy tắc
- **Không hardcode chuỗi UI.** Dùng `const { t } = useTranslation()` và các key:
  `t('user.name')`. Thêm key vào **mọi** file locale.
- Namespace nhóm các key (`nav.*`, `action.*`, `user.*`). Giữ key ổn định và mô tả rõ.
- Nội suy (interpolation): `t('greeting', { name })` với `"greeting": "Hi {{name}}"`.
- Số nhiều (pluralization): dùng key số nhiều của i18next (`key_one` / `key_other`).

## Locale của Ant Design
`ThemeProvider` chuyển `ConfigProvider locale` của AntD (vi_VN / en_US) đồng bộ với
ngôn ngữ đang chọn, để phần chữ của chính AntD (DatePicker, phân trang, v.v.) cũng
được bản địa hóa.

## Ngày / số
Dùng `Intl.*` (xem `formatDate` trong `shared/lib/utils.ts`) hoặc dayjs với đúng
locale. Đừng định dạng thủ công.

## Chuyển ngôn ngữ
Nút trên header gọi `i18n.changeLanguage('vi' | 'en')`. Mặc định là `vi`, dự phòng
(fallback) là `en`.
